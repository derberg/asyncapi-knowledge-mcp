# Deploy to Vercel (one-time setup)

This document describes the one-time setup needed to get
`derberg/asyncapi-knowledge-mcp` running on Vercel.

## 1. Create the Vercel project

Option A — via the Vercel CLI:

    vercel link   # follow prompts; choose "create new project"; name it asyncapi-knowledge-mcp

Option B — via the Vercel dashboard:
1. Go to https://vercel.com/new
2. Import `derberg/asyncapi-knowledge-mcp` from GitHub.
3. Framework preset: **Other** (the repo ships `vercel.json` with `"framework": null`).
4. Leave Build Command and Output Directory at their defaults (`vercel.json` sets them).
5. Click **Deploy**.

`vercel.json` already configures the project correctly:

```json
{
  "framework": null,
  "buildCommand": "",
  "outputDirectory": "chat",
  "functions": {
    "api/**/*.ts": { "includeFiles": "data/**" }
  }
}
```

The `includeFiles` directive bundles `data/vectors.bin` and `data/vectors.meta.json`
into each serverless function — the MCP and chat handlers read them at cold-start.

## 2. Set the environment variable

In the Vercel project settings (Settings → Environment Variables), add:

| Name | Value | Environments |
|---|---|---|
| `AI_GATEWAY_API_KEY` | your Vercel AI Gateway key | Production, Preview, Development |

The key is used by `api/chat.ts` (inference via `gpt-5-nano`) and by `scripts/embed.mjs`
during the weekly refresh.

## 3. Enable git auto-deploy

Vercel enables auto-deploy on the default branch (`main`) by default. No extra configuration
is needed — every push to `main` (including the weekly refresh commit) triggers a new
production deployment automatically.

> **Note:** the initial auto-deploy happens before the first search index exists
> (`data/` only contains `.gitkeep`), so `/api/mcp` and the chat will return errors
> until step 5 (first index build) lands on `main`.

## 4. Add the GitHub repository secret for refresh.yml

The weekly refresh workflow (`refresh.yml`) needs `AI_GATEWAY_API_KEY` to regenerate
`data/vectors.*`. Add it as a repository secret:

1. Go to the GitHub repo → Settings → Secrets and variables → Actions.
2. Click **New repository secret**.
3. Name: `AI_GATEWAY_API_KEY`, value: same key as above.

## 5. Build the first index

On a fresh clone, `data/vectors.bin` and `data/vectors.meta.json` may not exist yet
(they are generated by `npm run embed` and then committed). To build the first index:

    # Preview scope (no API call):
    node scripts/embed.mjs --dry-run

    # Full build (calls the Vercel AI Gateway):
    AI_GATEWAY_API_KEY=your-key npm run embed

    # Commit the generated data/ artifacts:
    git add data/
    git commit -m "chore: initial vector index"
    git push

Or trigger the refresh workflow manually from the Actions tab — it runs the full pipeline
and commits the result.

## 6. Verify the deployment

Once Vercel has deployed, the MCP server is available at:

    https://<your-domain>/api/mcp

Test it with the smoke test:

    MCP_URL=https://<your-domain>/api/mcp uv run --with mcp python tests/smoke_http.py

## 7. Update the placeholder domain

The placeholder domain `asyncapi-knowledge-mcp.vercel.app` appears in several places.
Once the real domain is confirmed, update it in:

- `plugins/asyncapi-knowledge/.mcp.json` — the `url` field for the MCP endpoint
- `chat/index.html` — any hardcoded domain references in the "use it in your own tools" panel
- `README.md` — the "Hosted MCP server" section

The plugin `.mcp.json` is the most important: it determines which endpoint Claude Code
uses when the plugin is installed.

## 8. Custom domain (optional)

In the Vercel project settings, go to Domains and add a custom domain. After DNS propagates,
update the placeholder domain references listed in step 7.

## PyPI trusted publishing setup (separate)

Publishing to PyPI via `publish-pypi.yml` requires a one-time trusted publisher setup on
pypi.org:

1. Go to https://pypi.org → Account settings → Publishing → Add a new pending publisher.
2. Project name: `asyncapi-knowledge-mcp`
3. GitHub repository owner: `derberg`
4. GitHub repository name: `asyncapi-knowledge-mcp`
5. Workflow filename: `publish-pypi.yml`
6. Environment: *(leave blank)*

After the first release, the package will appear at https://pypi.org/project/asyncapi-knowledge-mcp/.
