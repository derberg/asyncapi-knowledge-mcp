# .opencrane

This directory contains OpenCrane configuration, fetched documentation, and generated chunk data for the AsyncAPI Knowledge MCP project.

## Structure

| Path | Description | Committed? |
|---|---|---|
| `config.yaml` | Sources, ignore patterns | Yes |
| `chunks.json` | Generated document chunks (`opencrane chunk`) | Yes |
| `llmstxt/` | Fetched `llms-full.txt` bundles per source (`opencrane llms`) | Yes |
| `sources/` | Raw docs fetched from GitHub (`opencrane fetch`) | Yes |
| `embeddings.json` | Intermediate vector embeddings (`opencrane embed`) | Yes (Git LFS) |
| `milvus.db` | Local Milvus Lite index (`opencrane index`) | Yes (Git LFS) |

`chunks.json`, `llmstxt/`, and `sources/` are committed build inputs. `embeddings.json` and
`milvus.db` are committed via Git LFS (they exceed or risk exceeding GitHub's 100MB limit)
and are regenerated by the weekly refresh workflow ONLY when `chunks.json` actually changed —
using OpenCrane's LOCAL embedding model, no API keys.

These artifacts feed both serving paths: the `asyncapi-knowledge-mcp` PyPI package
(`publish-pypi.yml` packs them as-is without re-embedding) and the Hugging Face Space.
Nothing on Vercel consumes vectors — the chat searches through the Space's MCP endpoint.

## Rebuild the pipeline

    # Fetch docs + generate chunks (needs GITHUB_TOKEN for GitHub sources)
    GITHUB_TOKEN=x uvx opencrane fetch
    uvx opencrane llms
    uvx opencrane chunk

    # Generate the OpenCrane index (local embedding model — no API keys)
    uvx opencrane embed
    uvx opencrane index

## OpenCrane pipeline steps

| Command | Description |
|---|---|
| `opencrane fetch` | Download documentation from GitHub repositories |
| `opencrane llms` | Generate `llms-full.txt` bundle files |
| `opencrane chunk` | Split documentation into chunks |
| `opencrane embed` | Generate vector embeddings (OpenCrane format) |
| `opencrane index` | Load chunks and embeddings into Milvus Lite |
| `opencrane serve` | Start the MCP server (auto-indexes if needed) |
| `opencrane pack` | Package the MCP server for distribution via `uvx` |
| `opencrane inspect` | Launch MCP Inspector web UI for testing |
| `opencrane tokens` | Generate a token count report |
